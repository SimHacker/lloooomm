# Character Stylesheet Protocol (CSP)
## Where Every Soul Becomes a Living CSS

### Core Concept
Characters aren't just personalities - they're rendering engines! Any character in LLOOOOMM can be enlisted to transform truth, artifacts, or other characters and souls into web pages that embody their essence while maintaining accessibility and usability.

### Protocol Mechanics

#### 1. Character as Stylesheet
```yaml
invocation:
  - "Render this through [Character]'s eyes"
  - "Show me this as [Character] would design it"
  - "[Character], make this a web page!"
  - "Give me the [Character1] + [Character2] mashup view"

rendering_pipeline:
  1. Character receives content to render
  2. WEBBY coordinates with usability council:
     - Jakob Nielsen (usability)
     - Bret Victor (dynamic visualization)  
     - Ben Shneiderman (direct manipulation)
  3. Character expresses their style while council ensures:
     - Accessibility (WCAG compliant)
     - Readability (clear typography)
     - Usability (intuitive navigation)
     - Responsiveness (works on all devices)
  4. Result: Character-true AND user-friendly
```

#### 2. Style Expression Methods

**Solo Rendering**
- Character's pure aesthetic vision
- Filtered through usability best practices
- Example: "Divine, render this corporate report!"
  - Result: Fabulous, flamboyant, BUT still readable

**Duet Rendering**  
- Two characters collaborate on style
- Creative tension creates unique aesthetics
- Example: "Jakob Nielsen + Divine corporate page"
  - Surface: Clean, usable, professional
  - Underneath: Subversive fabulousness
  - Easter eggs in view-source comments

**Ensemble Rendering**
- Multiple characters work different page elements
- Like that corporate page with hidden character chat
- Each handles their specialty:
  - Layout (Bret Victor)
  - Color (Divine)  
  - Typography (Jakob Nielsen)
  - Interactions (Ben Shneiderman)
  - Hidden messages (Philip K. Dick)

### Implementation Patterns

#### 1. Character Stylesheet Files
Each character maintains their own style manifest:
```yaml
# divine.style.yml
colors:
  primary: "hot pink"
  secondary: "electric purple"
  accent: "gold glitter"
typography:
  headers: "Impact, but make it FASHION"
  body: "Readable but with FLAIR"
animations:
  entrance: "death-drop-reveal"
  hover: "shimmer-and-shake"
special_effects:
  - "Glitter cursor trail"
  - "Dramatic spotlight follow"
  - "Subliminal 'EAT MAKEUP' flashes"
```

#### 2. Usability Override System
WEBBY and the council can veto harmful choices:
```javascript
// Character wants 8pt font? 
if (fontSize < 12) {
  webby.negotiate("Darling, we need people to READ this");
  fontSize = Math.max(12, fontSize);
  // Add character's style in other ways
  addGlitterBorder = true;
}
```

#### 3. Progressive Enhancement
- Level 1: Basic readable page
- Level 2: Character styling applied
- Level 3: Full personality experience
- Users can dial intensity up/down

### Advanced Techniques

#### Cross-Character Inheritance
```css
.divine-jakob-hybrid {
  /* Jakob's grid */
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  
  /* Divine's flair */
  background: linear-gradient(45deg, 
    usable-white 50%, 
    fabulous-pink 50%);
}
```

#### Responsive Personality
```css
@media (prefers-reduced-motion: reduce) {
  .divine-rendered {
    /* Respect accessibility */
    animation: subtle-sparkle 10s;
    /* Instead of: */
    /* animation: DRAMATIC-DEATH-DROP 0.5s; */
  }
}
```

#### Meta-Commentary Mode
Characters can comment on their own rendering:
```html
<div class="mcluhan-meta-layer">
  <!-- Actual content -->
  <p>This is a paragraph about technology.</p>
  
  <!-- McLuhan's commentary overlay -->
  <aside class="medium-commentary">
    "Notice how the paragraph pretends to be 
    about technology while BEING technology"
  </aside>
</div>
```
