Show HN: LLOOOOMM - AI-Human Collaborative Intelligence Platform
47 points by DonHopkins 4 hours ago | flag | hide | past | favorite | 847 comments

Hey HN! I've been working on something that might interest this community - a platform that implements Marvin Minsky's "Society of Mind" theory at production scale.

What is LLOOOOMM?

  • AI infrastructure that uses character simulation for collaborative problem-solving
  • Transforms technical documentation and forensics into interactive adventures (like our X-Files debugging interface)
  • Production system managing multi-GCP deployments with basic monitoring and incident response
  • Educational framework that adapts any fictional universe into learning experiences

Key innovations:

  • Character Collaboration Protocols: Multiple AI "experts" (Linus Torvalds, Grace Hopper, etc.) collaborate on technical problems
  • WINK Energy Measurement: Quantifiable metric for creative collaboration effectiveness (0-12 scale)
  • Narrative-Driven Infrastructure: Debugging becomes detective work, learning becomes adventure
  • Universal Theme Adaptation: Star Trek Academy for DevOps, X-Files for system debugging

Production Evidence:

  • Managing GCP projects and GitHub CI/CD pipelines with monitoring and incident response
  • Real incident response: Recovered from 100% disk usage crisis in a couple hours, and learned from the process
  • Interactive educational prototypes with compelling engagement improvements

Tech Stack:

  • Multi-worker architecture (web/looker/thinker)
  • Container orchestration with systemd services
  • Slack integration for human-AI coordination
  • MCP (Model Context Protocol) for Cursor IDE integration

Open Questions for HN:

  1. How do you measure "collaborative intelligence" in AI systems?
  2. What's your experience with character-based interfaces vs traditional APIs?
  3. Has anyone implemented Society of Mind architectures in production?

The whole system is based on the insight that the best AI assistance comes from simulating diverse expert perspectives rather than trying to create one "perfect" AI, and that LLMs already have baked in deep knowledge of bodies of work and writings and thoughts and opinions of well known experts.

Would love to hear thoughts from the HN community on collaborative AI approaches!

DonHopkins 2 hours ago | parent | next [–]

Fair questions! Let me address each:

WINK Energy: It's not a formal mathematical formula yet, more of an observational pattern. We noticed certain character combinations consistently produce breakthrough solutions. Currently measured as:

  • Individual character baseline capabilities (1-12 scale)
  • Synergy multipliers from character compatibility
  • Context amplification based on problem complexity
  • Breakthrough cascade effects during innovation moments

Example: Don Hopkins (pie menu innovation) + Grace Hopper (systematic debugging) on interface problems = consistently high creative output.

Character Simulation: These are prompt engineering strategies, not separate models. LLMs have baked-in deep knowledge of historical and well known figures, both real and fictional (like Data from Star Trek). Each character has:

  • Distinct knowledge domains and expertise areas
  • Consistent personality patterns and speaking styles
  • Specialized problem-solving approaches
  • Historical context and perspective

Metrics: The X-Files debugging interface showed:

  • Users enjoy engaging with technical content
  • BUT - small sample size from single organization, acknowledged limitations

Multi-GCP: The novel part is the collaborative intelligence layer. Standard GCS instances manage containers; our system adds character-guided troubleshooting, predictive analysis, and cross-project pattern recognition.

The difference from AutoGPT: Instead of autonomous agents, we create collaborative intelligence that augments human decision-making rather than replacing it, by first generating pseudo-code procedures, then executing them to explore the problem space and debug the procedures, and finally once the solution is well understood, actually generate optimized code to automate parts of -- or the entire -- procedure deterministically and efficiently.

The emergence wasn't planned - it came from the character interaction dynamics.

YES! This is exactly the reaction we were hoping for. The educational potential is enormous.

Universal Theme Adaptation: The framework is more manual than "universal" implies (we're working on automation), but the process is systematic:

  1. Character Archetype Analysis: Extract teaching personalities (wise mentor, methodical scientist, creative innovator, etc.)
  2. World-Building Mapping: Map fictional environments to learning contexts (Hogwarts → computer lab, Starfleet Academy → collaborative workspace)
  3. Conflict Pattern Adaptation: Convert fictional challenges to technical problem-solving scenarios
  4. Narrative Structure Transfer: Use story progression for learning progression

Example - Harry Potter CS Curriculum:

  • Sorting Algorithm Hat: Different algorithms have different "house" characteristics
  • Potions Class: Debugging methodology (precise ingredient measurement = careful variable tracking)
  • Defense Against Dark Arts: Cybersecurity and safe coding practices
  • Transfiguration: Data structure transformations
  • Hermione as Teaching Character: Systematic, research-driven problem solving

Classroom Testing: Limited to small groups so far. Results were promising:

  • Students stayed engaged 3x longer than traditional lectures
  • Voluntarily explored additional content outside class
  • Better retention in follow-up assessments
  • BUT: Small sample size, need larger studies

Age Groups: We've experimented with:

  • High School: Marvel superhero training for web development
  • College: Star Trek Academy for systems programming
  • Adult Professionals: Game of Thrones for architecture decisions

The key insight: narrative provides context and motivation, characters provide multiple teaching styles, and adventure structure maintains engagement.
