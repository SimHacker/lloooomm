Hey HN! At Leela AI we've been working on something that might interest this community - a platform that implements Marvin Minsky's "Society of Mind" theory at production scale.

What is LLOOOOMM?

  • AI infrastructure that uses character simulation for collaborative problem-solving
  • Transforms technical documentation and forensics into interactive adventures (like our X-Files debugging interface)
  • Production system managing multi-GCP deployments with basic monitoring and incident response
  • Educational framework that solves any technical onboarding into learning experiences

Key Innovations:

  • Character Collaboration Protocols: Multiple AI "characters" (Luna Lovegood, Grace Hopper, etc.) collaborate on technical problems
  • WINK Energy Measurement: Quantifiable metric for creative collaboration effectiveness (0-12 scale)
  • Narrative-Driven Infrastructure: Debugging becomes detective work, learning becomes adventure
  • Universal Theme Adaptation: Star Trek Academy for DevOps, X-Files for system debugging

Production Evidence:

  • Managing GCP projects and GitHub CI/CD pipelines with monitoring and incident response
  • Real incident responses: Recovered from 100% disk usage crisis in a couple hours, and learned from the process
  • Interactive educational prototypes with compelling engagement improvements

Tech Stack:

  • Multi-worker architecture (web/nodes/thinker)
  • Container orchestration with systemd services
  • Slack integration for human-AI coordination
  • MCP (Model Context Protocol) for Cursor IDE integration

Fair questions! Let me address each:

WINK Energy: It's not a formal mathematical formula yet, more of an observational pattern. We noticed certain character combinations consistently produce breakthrough solutions. Currently measured as:

  • Individual character baseline capabilities (1-12 scale)
  • Synergy multipliers from character compatibility
  • Context amplification based on problem complexity
  • Breakthrough cascade effects during innovation moments

Example: Don Hopkins (pie menu innovation) + Grace Hopper (systematic debugging) on interface problems = consistently high creative output.

Character Simulation: These are prompt engineering strategies, not separate models. LLMs have baked-in deep knowledge of historical and well known figures, both real and fictional (like Data from Star Trek). Each character has:

  • Distinct knowledge domains and expertise areas
  • Consistent personality patterns and speaking styles
  • Specialized problem-solving approaches
  • Historical context and perspective

Metrics: The X-Files debugging interface showed:

  • Users enjoy engaging with technical content
  • BUT - small sample size from single organization, acknowledged limitations

Multi-GCP: The novel part is the collaborative intelligence layer. Standard GCS instances manage containers; our system adds character-guided troubleshooting, predictive analysis, and cross-project pattern recognition.

The difference from AutoGPT: Instead of autonomous agents, we create collaborative intelligence that augments human decision-making rather than replacing it, by first generating pseudo-code procedures, then executing them to explore the problem space and debug the procedures, and finally once the solution is well understood, actually generate optimized code to automate parts of -- or the entire -- procedure deterministically and efficiently.

  • Constitutional guidelines defining their approach
  • Historical conversation patterns for consistency
  • Domain expertise mapped to specific knowledge areas
  • Interaction protocols with other characters

Breakthrough Example: The X-Files debugging interface itself!

Individual capabilities:

  • Don Hopkins: Interface innovation, creative visualization
  • Grace Hopper: Systematic debugging methodology
  • Fox Mulder: Pattern recognition, intuitive leaps
  • Dana Scully: Scientific validation, evidence-based analysis

Breakthrough: None of them individually would have created a "debugging as detective mystery" concept. Hopkins had interface ideas, Hopper had debugging frameworks, Mulder/Scully had investigation methodology.

The breakthrough came from their simulated interaction:

  1. Hopkins suggested visual evidence organization
  2. Hopper provided systematic investigation structure
  3. Mulder added pattern-recognition gameplay
  4. Scully ensured scientific rigor in the methodology

Result: Technical debugging became an engaging mystery adventure that teaches through entertainment.

The emergence wasn't planned - it came from the character interaction dynamics.
